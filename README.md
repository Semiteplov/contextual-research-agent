# Интеллектуальный многоагентный ассистент для научно-технической деятельности

Веб-сервис для анализа научных публикаций в области машинного обучения, основанный на многоагентной RAG-архитектуре с графом знаний, адаптивной маршрутизацией запросов и пост-проверкой ответов.

---

## Возможности

Система автоматизирует шесть типовых аналитических задач работы с научной литературой:

- **factual_qa** — поиск фактов в статье с обязательным цитированием источников
- **summarization** — структурированное резюме статьи или метода
- **comparison** — сравнение методов по согласованным критериям
- **critical_review** — критический анализ методологических ограничений
- **methodological_audit** — проверка корректности экспериментальной методологии
- **idea_generation** — формулирование гипотез для дальнейшей работы

Каждая задача обрабатывается через специализированный промпт Generator Agent с учётом извлечённого контекста.

---

## Архитектура

Система состоит из пяти основных компонент:

```
┌─────────────────────────────────────────────────────────┐
│  Gradio UI (чат, debug-панель, live-трассировка)        │
└─────────────────────────────────────────────────────────┘
                          │
┌─────────────────────────────────────────────────────────┐
│  FastAPI: REST sync, SSE streaming, WebSocket trace     │
└─────────────────────────────────────────────────────────┘
                          │
┌─────────────────────────────────────────────────────────┐
│  Agent (LangGraph):                                     │
│  Router → Planner → Retriever → Generator → Critic →    │
│           ParallelExecutor       Synthesizer            │
└─────────────────────────────────────────────────────────┘
            │              │              │
       ┌────▼────┐    ┌────▼─────┐   ┌────▼────┐
       │ Qdrant  │    │PostgreSQL│   │ Ollama  │
       │(dense + │    │(citation │   │  (LLM,  │
       │ sparse) │    │  graph)  │   │ embed)  │
       └─────────┘    └──────────┘   └─────────┘
```

### Многоагентная оркестрация (LangGraph)

| Агент | Роль |
|---|---|
| **Router** | Классифицирует запрос: intent (mode) + complexity (simple/multi-aspect/complex) |
| **Planner** | Декомпозирует составные запросы на независимые подзапросы |
| **Retriever** | Многоканальный поиск по 5 каналам (dense, sparse, citation graph, entity graph, paper-level) |
| **Generator** | Генерация ответа в нужном когнитивном режиме |
| **ParallelExecutor** | Параллельное исполнение независимых подзапросов через `asyncio.gather` |
| **Critic** | Пост-проверка фактической достоверности с retry-механизмом (бюджет 1 попытка) |
| **Synthesizer** | Объединение частичных ответов в единый связный |

### Информационный поиск

Пятиканальная архитектура с агрегацией через Reciprocal Rank Fusion (RRF) и cross-encoder переранжированием:

- **Dense retrieval** — Qwen3-Embedding-0.6B (1024-dim) через Qdrant
- **Sparse retrieval** — BM25 через Qdrant sparse vectors
- **Citation graph** — рекурсивный обход цитатных связей в PostgreSQL
- **Entity graph** — поиск через общие научные сущности (методы, датасеты, задачи)
- **Paper-level** — поиск на уровне статьи с расширением до чанков

## Технологический стек

| Категория | Компоненты |
|---|---|
| **Языковые модели** | Qwen3-8B, Qwen3-4B, Qwen3-Embedding-0.6B |

| **Inference** | Ollama |

| **Оркестрация агентов** | LangGraph |
| **Серверная часть** | FastAPI, uvicorn |
| **Пользовательский интерфейс** | Gradio 5.x |
| **PEFT-эксперименты** | Unsloth, PEFT 0.13, llama.cpp (GGUF) |
| **Отслеживание экспериментов** | MLflow |
| **Оценка качества** | LLM-as-judge (GPT-5.4-mini через OpenRouter) |
